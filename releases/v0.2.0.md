# AgentPay SDK version 0.2.0 release notes

**Release date:** 2026-03-27  
**Current version:** v0.2.0

These notes cover what is new in AgentPay SDK `v0.2.0` relative to `v0.1.0`.

For the baseline local-first runtime, wallet setup, backup and restore, policy model, installer flow, and Bitrefill integration, see [v0.1.0 release notes](./v0.1.0.md).

## Release summary

AgentPay SDK `v0.2.0` adds HTTP payment flows for `x402` and `mpp`, expands EIP-3009 signing support, and adds Tempo mainnet plus Tempo session-based MPP workflows.

## What's new in v0.2.0

### x402

- added `agentpay x402 <url>`
- supports exact-payment and EIP-3009 x402 HTTP payment flows
- supports reusable HTTP request controls including `--method`, repeatable `--header`, `--data`, and `--json-body`

### MPP

- added `agentpay mpp <url>`
- supports MPP HTTP 402 payments on EVM-compatible chains
- supports Tempo session flows including session open, voucher signing, optional `--deposit`, automatic top-up, and explicit close
- added persisted session reuse via `--session-state-file`
- added decoded `Payment-Receipt` output plus JSON and NDJSON automation modes

### Chain and signing support

- added Tempo mainnet support with chain ID `4217`
- added Rust agent CLI support for EIP-3009 transfer and receive authorization signing requests
- the local signer path now covers `transfer`, `transfer-native`, `approve`, `broadcast`, `x402`, and `mpp`
